# Arab-Islam Network DB Project (AINet-DB)

本リポジトリは、アル＝サハーウィー(al-Sakhāwī, d. 902 H / 1497 CE)の『輝く光(*al-Ḍawʾ al-Lāmiʿ li-Ahl al-Qarn al-Tāsiʿ*)』を構造化し、マムルーク朝後期(9世紀ヒジュラ)の知識ネットワークを再構築するためのデジタル人文学プロジェクトです。

## プロジェクトの概要

- **対象資料**: 9世紀ヒジュラ暦の人物録(約 13,000 名の伝記エントリを収録)
- **手法**: 生成 AI(Gemini / Claude)による構造化抽出 + 専門家による校閲
- **目標**: TEI XML 形式でのデータベース化、Wikidata / GeoNames 等の外部権威ファイルとの連携、人的ネットワークの可視化

---

## プロジェクトのデータセット

| ファイル / リソース | 概要 |
|---|---|
| **`0_DawForAIND_Cleaned.txt`** | 『輝く光』の本文(クリーン化済み、AIND-D ID 付与済み)。アプリへの入力にはこのテキストを使用 |
| **ID-Master**(Google スプレッドシート) | 地名・人物・機関・書物・学問分野などの ID を一元管理するマスターリスト。解析時にアプリが自動参照 |
| **Sakhawi_PersonList**(Google スプレッドシート) | 『輝く光』に立項されている人物の一覧。担当者の進捗管理・解析結果のレビューに使用 |

### 本文テキストの形式

`0_DawForAIND_Cleaned.txt` は次の形式で各エントリを含みます:

```
###$AIND-DXXXXX | NNNNNNNNNNNN$# {マーカー} 本文…
```

- **AIND-DXXXXX**: AIND-D ID(5 桁、xml:id として使用)
- **NNNNNNNNNNNN**: 12 桁数字の元データ ID(source 属性として使用)
- **マーカー**:
  - `$` 男性立項
  - `$$` 女性立項
  - `$$$` 相互参照(他の項目への言及)
  - `|` `||` 補助マーカー(複合エントリ等)

例:
```
###$AIND-D00001 | 932540579843$# $ آدم بن سعد بن عيسى الكيلاني الأصل ثم المكي
```

全 15,530 エントリ、うち立項 13,440 件。

## 識別子の体系(本プロジェクトの中核)

本プロジェクトでは **2 つの主要識別子** を使い分けます。

| 識別子 | 形式 | 例 | 役割 | データソース |
|---|---|---|---|---|
| **AIND-D ID** | `AIND-DXXXXX`(5 桁) | `AIND-D00001` | XML の `xml:id` 属性として使用、TEI の不変キー | テキストヘッダーから取得 |
| **12 digits ID** | 12 桁数字 | `932540579843` | XML の `source` 属性として使用、元データの自然キー | テキストヘッダーから取得 |

### XMLでの使われ方

```xml
<person xml:id="AIND-D00001" source="932540579843">
    <persName type="full" xml:lang="ar">آدم بن سعد بن عيسى الكيلاني...</persName>
    ...
</person>
```

- **xml:id**: TEI 内部での参照キー。他のエントリから `passive="#AIND-D00001"` のように参照される
- **source**: 元データへの参照。データの追跡可能性を確保

### XMLファイル名の規約

ダウンロード XML のファイル名は次の形式:

```
{AIND-D ID}_{12 digits ID}.xml
```

例: `AIND-D00001_932540579843.xml`

# AINet-DB Researcher Pro 使い方ガイド

**対象**: デジタルツールに不慣れな人文学研究者・アシスタント

**目的**: イスラーム伝記史料から人物データを抽出し、データベース用の XML ファイルを作成する

## このツールでできること

アラビア語の人名録(伝記史料)のテキストを貼り付けると、AI が自動的に人物情報を読み取り、整理してくれます。研究者はその結果を確認・修正しながら、学術データベース用のフォーマット(TEI-XML)を作成できます。また、作成したデータは共有のスプレッドシートに保存することができます。

## 画面の構成

画面は大きく **左側(サイドバー)** と **右側(メインエリア)** に分かれています。

- **左側**: 史料テキストの入力・解析・翻訳の表示
- **右側**: AI が抽出したデータの確認・修正・XML 出力・保存

メインエリアは縦に長いフォームになっており、上から順に以下のセクションが並びます。

1. **基本情報**(AIND-D ID、12 digits ID、性別、persName 等)
2. **ニスバ・ラカブ**(別名)
3. **生没年**
4. **法学派(Madhhab)・スーフィー教団**
5. **🎓 Teachers**(師匠)
6. **🧑‍🎓 Students**(弟子)
7. **📍 Activities**(地理的活動)
8. **🏛️ Institutions**(所属機関)
9. **🏅 Offices**(役職)
10. **👨‍👩‍👧 Family Relations**(家族関係)
11. **🤝 Social Relations**(社会関係)
12. **📅 Biographical Events**(伝記的出来事)
13. **メモ**(Person Notes / Editors' Notes / 翻訳)
14. **🖊️ respStmt**(作業者情報)

## 作業の流れ

### ▶ ステップ 1: 史料テキストを入力する

画面左側(サイドバー)の「Source Text」エリアに、解析したい人物の伝記テキストをアラビア語で貼り付けます。

**重要**: 一度に入力するのは **人物 1 人分** のテキストにしてください。複数人を同時に入力すると、AI が混乱して正しく抽出できません。

テキストには `###$AIND-DXXXXX | NNNNNNNNNNNN$#` のヘッダーを含めてください。これによりアプリが AIND-D ID と 12 digits ID を自動抽出します。

### ▶ ステップ 2: 解析する

「🔍 解析する」ボタンを押すと、Gemini API が起動して伝記テキストを解析します。数秒〜数十秒待つと、メインエリアの各フィールドに自動的に抽出結果が反映されます。

### ▶ ステップ 3: データを確認・修正する

AI が抽出した内容を、研究者の目で確認します。間違いや漏れがあれば、各フィールドを直接編集して修正します。

**便利機能**:

- **▲▼ ボタン**: Teachers / Students / Activities / Institutions / Offices などの各エントリを上下に並べ替えできます。時系列順に整理する際に便利です。
- **❌ ボタン**: 不要なエントリを削除します。
- **＋ add ボタン**: 新しいエントリを追加します。
- **🗑️ 入力をクリア ボタン**: 画面右上にあり、入力内容を一括でクリアできます。誤クリック防止のため 2 段階確認があります。担当者名は保持されます。
- **↗ 翻字を一括補完 ボタン**: 空のラテン欄(ラテン文字翻字)を IJMES 規格で一括補完します(既存の入力は保持)。
- **🔎 ID-Master 照合 ボタン**: アラビア語の名前を ID-Master と完全一致照合し、未確定の ID 欄に自動で値を入れます。
- **🔢 採番を更新 ボタン**: プレースホルダー(`TMP-X-00000` 等)を空き番号で自動採番します。

### ▶ ステップ 4: XML を確認・ダウンロードする

メインエリアの「TEI-XML Export」セクションに自動生成された XML が表示されます。

- **📥 XML をダウンロード ボタン**: `{AIND-D ID}_{12 digits ID}.xml` 形式のファイルとして 1 クリックで保存できます。
- **📋 XML をクリップボードにコピー ボタン**: XML 全体をクリップボードに保存します。

### ▶ ステップ 5: スプレッドシートに保存する

最終的なデータは「📤 スプレッドシートに保存」ボタンで共有スプレッドシート(Sakhawi_PersonList)に保存できます。担当者名(Assignee)を選んでからボタンを押してください。

## 入力フィールドの詳細

### 基本情報

| フィールド | 説明 | 例 |
|---|---|---|
| **AIND-D ID (xml:id)** | テキストヘッダーから自動抽出。XML の xml:id として使用 | `AIND-D00001` |
| **12 digits ID (@source)** | テキストヘッダーから自動抽出。XML の source 属性として使用 | `932540579843` |
| **Sex** | 性別 | M(男性)/ F(女性)/ U(不明) |
| **persName (Full Arabic)** | 法学派まで含めたフルネーム | إبراهيم بن أحمد ... الشافعي |
| **persName (Ism/Father/GF)** | 本名 + 父 + 祖父 | إبراهيم بن أحمد بن عبد الكافي |

### 生没年

ヒジュラ暦の年だけでなく、年月日も入力できます。確実性と注記、推論根拠も記録できます。

| フィールド | 説明 | 例 |
|---|---|---|
| Birth (H) / Death (H) | ヒジュラ暦 | `850` / `850-09` / `850-09-15` |
| Birth (G) / Death (G) | 西暦(自動計算) | `1446` / `1446-12` / `1446-12-21` |
| **Cert** | 確実性 | high / medium / low / 未指定 |
| **Note** | 注記 | "Ca. 850" / "異説あり(901)" |
| **Inference Note** | 推論の根拠(英語推奨) | 'Inferred from "died at 50 in 900 H".' |

### Teachers / Students

各エントリは 4 行構成です。

- **1 行目**: Name(師匠/弟子の名前)+ Person ID
- **2 行目**: 
  - **Method**(学習方法): 動詞(سمع, أخذ, قرأ 等)。プルダウン or 自由記述
  - **Field**(学習分野): 学問分野(الفقه, الحديث 等)。プルダウン or 自由記述
- **3 行目**: Text Arabic / Text Latinized / Text ID(学んだ書物)
- **4 行目**: Date / Place(学んだ年・場所)

▲▼ ボタンで時系列順に並べ替えできます。

### Activities

地理的活動(移住・訪問・巡礼など)を記録します。Type:

| Type | 説明 |
|---|---|
| residence | 居住 |
| visit / travel | 訪問・移動 |
| study | 遊学 |
| **hajj / umrah** | 大巡礼・小巡礼 |
| jāwara | メッカ/メディナでの長期滞在(مجاورة) |
| riḥla | 学問的旅行(الرحلة) |
| buried | 埋葬 |
| legacy | 死後の遺産・影響 |
| other | その他 |

### Family Relations

家族関係を記録します。relation の種類は以下の 16 種類:

- **直系**: father, mother, son, daughter
- **横系**: brother, sister, spouse
- **祖父母**: grandfather, grandmother
- **傍系**: uncle, aunt, cousin, siblings_child(甥・姪)
- **系譜**: ancestor(先祖), descendant(子孫)
- **その他**: other

**重要**: Family は **史料に明示的に言及された関係のみ** 記録します。nasab 連鎖(`محمد بن أحمد` から父・祖父を自動展開)はしません。kunyah(`أبو فلان`)から息子を推測することもしません。

### Biographical Events

地理的移動を伴わない人生のイベント(著作・政治事件・宗教的事件など)を記録します。

| Type | 説明 | 例 |
|---|---|---|
| political | 政治的事件 | 出陣、政争への関与 |
| cultural | 著作・知的活動 | 著作、講義 |
| religious | 宗教的事件 | 葬礼、霊的体験 |
| other | その他 | 疫病、自然災害 |

**重要**: ハッジ(巡礼)は Activities に記録します。Bio Events には含めません。

### Social Relations

家族でも師弟でもない社会的関係を記録します。

| Type | 説明 |
|---|---|
| patron | 庇護者 |
| client | 被庇護者 |
| colleague | 同僚 |
| rival | 論敵 |
| friend | 友人 |
| correspondent | 書簡相手 |
| successor / predecessor | 後継者・前任者 |
| other | その他 |

### respStmt(作業者情報)

この `person` 要素の編集に関わった担当者を記録します。複数の作業履歴を並列で記録可能。役割の選択肢:

- 初版作成
- 修正・追記
- 校閲
- 翻訳
- ID 照合
- その他

## ID の体系

このツールでは、人物・地名・機関などに外部データベースの ID を付与します。

| 種類 | 用途 | 例 |
|---|---|---|
| **GeoNames ID**(数字のみ) | 地名 | `104515`(メッカ) |
| **Wikidata Q-ID**(`Q` 始まり) | 概念・組織・人物・書物 | `Q160851`(ハナフィー派) |
| **TMP-P-**(仮 ID、6 桁) | 外部 ID 未確定の人物 | `TMP-P-000001` |
| **TMP-N-**(仮 ID、5 桁) | ニスバ | `TMP-N-00001` |
| **TMP-L-**(仮 ID、5 桁) | 地名 | `TMP-L-00001` |
| **TMP-I-**(仮 ID、5 桁) | 機関 | `TMP-I-00001` |
| **TMP-O-**(仮 ID、5 桁) | 役職 | `TMP-O-00001` |
| **TMP-T-**(仮 ID、5 桁) | 書物・典籍 | `TMP-T-00001` |
| **TMP-S-**(仮 ID、5 桁) | 学習方法・分野 | `TMP-S-00001` |

### 出力 XML での参照形式

XML 内では次のプレフィックスで参照を区別します:

| プレフィックス | 種類 | 例 |
|---|---|---|
| `wd:` | Wikidata | `wd:Q160851` |
| `gn:` | GeoNames | `gn:104515` |
| `#` | プロジェクト内参照(TMP-, AIND-D) | `#TMP-P-000001`, `#AIND-D00033` |

## ID Master について

画面左側の「📋 ID Master 状態」を開くと、研究グループが共有している ID リストを確認できます。このリストに登録されている地名・機関・人物・学習方法・学習分野は、解析時に AI が自動的に正しい ID を割り当てます。

Method(学習方法)と Field(学習分野)のプルダウンは ID-Master から動的に語彙を読み込みます。ID-Master に新しい語彙が追加されると、アプリも自動で反映します。

### ID Master に新規 ID を登録する方法

史料を読んでいると、まだ ID Master に登録されていない地名・機関・人物・書物などが出てきます。そのような場合、外部データベースで ID を調べて ID Master に追加します。

#### ① 地名の場合 → GeoNames で調べる

1. https://www.geonames.org/ を開く
2. 地名を検索(英語かアラビア語)
3. ヒットした地名のページで URL の数字部分を取得(例: `https://www.geonames.org/104515/` → ID は `104515`)
4. ID Master スプレッドシートに追加:
   - Category: `Place`
   - Arabic: アラビア語表記
   - Latin: ラテン文字表記
   - ID: GeoNames の数字

#### ② 機関・概念・法学派・スーフィー教団の場合 → Wikidata で調べる

1. https://www.wikidata.org/ を開く
2. 名称で検索
3. ヒットした項目の URL から Q-ID を取得(例: `https://www.wikidata.org/wiki/Q160851` → ID は `Q160851`)
4. ID Master スプレッドシートに追加:
   - Category: `Institution` / `Concept` 等
   - ID: `Q` から始まる Wikidata の ID

#### ③ 人物の場合 → Wikidata で調べる

著名な人物(Ibn Ḥajar al-ʿAsqalānī = `Q471116` 等)は Wikidata に登録されています。同上の方法で Q-ID を取得。

#### ④ 書物・典籍の場合 → Wikidata で調べる

主要なハディース集等は Wikidata に登録があります:

| 書物 | Q-ID |
|---|---|
| al-Bukhārī の Ṣaḥīḥ | `Q208507` |
| Muslim の Ṣaḥīḥ | `Q193272` |
| al-Tirmidhī の Sunan | `Q1248893` |
| Abū Dāwūd の Sunan | `Q593290` |
| al-Nasāʾī の Sunan | `Q1140365` |
| Ibn Mājah の Sunan | `Q940817` |
| Mālik の Muwaṭṭaʾ | `Q900871` |
| al-Dāraquṭnī の Sunan | `Q12217063` |

#### ⑤ 見つからない場合は仮 ID のままでよい

外部データベースに該当が見つからない場合、TMP- 仮 ID(例: `TMP-P-000001`)のままで構いません。アプリの「🔢 採番を更新」ボタンで自動採番されます。

後日、正式な Wikidata 項目が作成されたら、ID Master を更新することで全エントリに反映されます。

## スプレッドシート運用

### Sakhawi_PersonList の列構成

| 列 | 内容 | アプリの動作 |
|---|---|---|
| **A** | 行数 | 触らない(関数式があれば保持) |
| **B** | 担当者 | 書き込む |
| **C** | AIND-D ID(`AIND-DXXXXX`) | **触らない**(手動記入領域) |
| **D** | 12 digits ID | **触らない**(検索キー、手動記入領域) |
| **E** | persName (Full Arabic) | 書き込む |
| **F** | persName (Ism/Father/GF) | 書き込む |
| **G** | Birth (H) | 書き込む |
| **H** | Death (H) | 書き込む |
| **I** | Madhhab | 書き込む |
| **J** | Editors' Notes | 書き込む |

### C列・D列を「触らない」運用の理由

- AIND-D ID と 12 digits ID は元データに由来する不変キーであり、プロジェクト管理者が管理する
- アプリが書き換えるとデータの整合性が崩れる可能性があるため、安全のため手動記入領域とする
- 新規行追加時も、アプリは B / E〜J 列のみに書き込み、C / D 列は空のまま残す(後で人間が記入)

### 既存行更新と新規行追加

- **既存行更新**: D 列(12 digits ID)で検索し、一致する行があれば B 列と E〜J 列を更新
- **新規行追加**: 一致する行がない場合、D 列が空の最初の行を探し、その行の B / E〜J 列に書き込む(C / D 列は手動記入のため空のまま)
  - 保存後、画面に「C 列・D 列に AIND-D ID と 12 digits ID を手動で記入してください」と表示される

## XML 出力

### 出力される XML の構造

```xml
<person xml:id="AIND-D00001" source="932540579843">
    <persName type="full" xml:lang="ar">آدم بن سعد بن عيسى الكيلاني...</persName>
    <persName type="name_only" xml:lang="ar">آدم بن سعد بن عيسى</persName>
    <persName type="nisbah" xml:lang="ar" ref="...">الكيلاني</persName>
    <persName type="laqab" xml:lang="ar">...</persName>
    <sex value="M"/>
    <affiliation type="madhhab" ref="wd:Q82245">Shafi'i</affiliation>
    <birth when-custom="0850" when="1446" cert="high"/>
    <death when-custom="0902" when="1497" cert="high"/>
    <listRelation>
        <relation type="personal" subtype="teacher" n="1" 
                  active="#{師匠のID}" passive="#AIND-D00001">
            <desc type="method" ref="..." xml:lang="ar">سمع</desc>
            <desc type="field" ref="..." xml:lang="ar">الحديث</desc>
        </relation>
        ...
    </listRelation>
    <event type="residence" when-custom="0870">
        <placeName ref="gn:104515">مكة</placeName>
    </event>
    ...
    <respStmt>
        <resp xml:lang="ja">初版作成</resp>
        <persName>Erina Ota</persName>
        <date when="2026-05-11"/>
    </respStmt>
</person>
```

### XML 出力の設計方針

- **xml:id は `AIND-DXXXXX` 形式**(5 桁)、TEI の不変キー
- **source 属性に 12 桁数字**を保持(元データへの参照を明示)
- **日付は `when-custom` でヒジュラ暦、`when` で西暦** を併記
- **言語タグは厳密に**: アラビア語 `xml:lang="ar"`、ラテン文字翻字 `xml:lang="ar-Latn"`、日本語 `ja`、英語 `en`
- **推論注記** `<note type="inference">` で推論の根拠を記録

## データ品質と運用方針

### 抽出時の重要原則

#### 1. nasab 連鎖から家族関係を自動展開しない

`محمد بن أحمد بن علي بن حسن` のような nasab(系譜)連鎖から、自動的に「父=أحمد、祖父=علي」のように展開してはいけません。Family Relations は **史料が明示的に言及した関係のみ** 記録します。nasab 自体は `full_name` / `name_only` に保存されています。

#### 2. ancestor のタイトルを subject に流用しない

`إبراهيم بن العلامة محمد` の場合、`العلامة`(博学者の意)は **父のタイトル** であり、subject(إبراهيم)のタイトルではありません。subject のタイトル(laqab/honorific)に組み入れないこと。

#### 3. 法学派は nisbah ではなく madhhab に

`الشافعي` は nisbah(地名・家系由来の名)ではなく madhhab(法学派)です。`madhhab` フィールドに記入し、nisbahs には記入しません。

#### 4. ハッジは「حج」「حجّ」と明記された場合のみ

`ذهب إلى مكة`(メッカに行った)は visit。`حج سنة كذا`(年に巡礼した)は hajj。曖昧な場合は visit を選びます。

#### 5. 集合代名詞は記録しない

`وغيره / جماعة / آخرون / غير واحد` 等は「その他多数」を意味する曖昧表現。これらは relation / teacher / student / family 等に記録しません。

#### 6. 史料の参照マーカー

`الآتي`(後述する人)や `الماضي / المتقدم`(前述した人)は、別の伝記エントリへの相互参照を示します。これらを見つけた場合は editors_notes に "Marked الآتي — cross-ref to a later entry." 等のメモを残します。

## 既知の問題と運用上の注意

### AIND-D ID 欄が空のまま

- 史料テキストに `###$AIND-DXXXXX | NNNNNNNNNNNN$#` ヘッダーが含まれていない可能性
- 解析後に手動で AIND-D ID と 12 digits ID 欄に入力できます
- ヘッダー形式が `###$AIND-D00001 | 932540579843$#` のように `|`(パイプ)で区切られているかを確認

### 12 digits ID が 12 桁にならない

- 12 桁の半角数字でない場合、警告が表示されます
- 全角数字や空白の混入に注意

### 解析ボタンを押しても何も起きない

- Gemini API のキーが secrets に設定されているか確認
- ネットワーク接続を確認

### AI の抽出結果が明らかに間違っている

- AI は完璧ではありません。必ず人間が確認します
- 明らかな誤りは手動で修正してください
- AI が頻繁に間違う部分があれば、Geminiプロンプトの調整が必要(管理者に連絡)

### スプレッドシートへの保存でエラーが出る

- secrets に `[gcp_service_account]` が登録されているか
- スプレッドシートがそのサービスアカウントに共有されているか

### スプレッドシートの C 列・D 列がアプリで更新されない

- これは **仕様** です。C 列(AIND-D ID)と D 列(12 digits ID)はアプリでは触らず、ユーザーが手動で記入する運用です
- 新規行追加時は、保存後にアプリが「手動で記入してください」と通知します

### ページを閉じたらデータが消えた

- アプリのデータはセッション中のみメモリに保持されます
- 作業終了時は必ず XML ダウンロードまたはスプレッドシート保存を行ってください
