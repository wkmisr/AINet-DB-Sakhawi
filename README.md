# Arab-Islam Network DB Project (AINet-DB)

本リポジトリは、アル＝サハウィーの『輝く光（al-Daw' al-Lami'）』を構造化し、マムルーク朝期の知識グラフを構築するためのプロジェクトです。

## プロジェクトの概要

- **対象**: 9世紀ヒジュラ暦の人物録（約13,000名）
- **手法**: 生成AI（Claude/Gemini）による抽出 ＋ 専門家による校閲
- **目標**: TEI形式による構造化および外部DB（Wikidata等）との連携

## プロジェクトのデータセット

- **DawForAIND.txt** — シャハーウィー著『輝く光（al-Ḍawʾ al-Lāmiʿ）』のデジタルテキスト。解析ツールへの入力にはこちらを使用します。最新版は `0__DawForAIND_step3_final_v3.txt`（v3.0で12桁IDベースに統一）です。
- **ID-Master** — 地名・機関・人物・書物などのIDを一元的に蓄積・管理するマスターリストです。解析時にAIが自動参照します。
- **Sakhawi_PersonList** — 『輝く光』に立項されている人物の一覧です。解析ツールで抽出したデータが自動的に反映されます。

# AINet-DB Researcher Pro 使い方ガイド

**対象**: デジタルツールに不慣れな人文学研究者の方へ
**目的**: イスラーム伝記史料から人物データを抽出し、データベース用のXMLファイルを作成する

## このツールでできること

アラビア語の人名録（伝記史料）のテキストを貼り付けると、AIが自動的に人物情報を読み取り、整理してくれます。研究者はその結果を確認・修正しながら、学術データベース用のフォーマット（TEI-XML）を作成できます。また、作成したデータは共有のスプレッドシートに保存することができます。

## 画面の構成

画面は大きく **左側（サイドバー）** と **右側（メインエリア）** に分かれています。

- **左側**: 史料テキストの入力・解析・翻訳の表示
- **右側**: AIが抽出したデータの確認・修正・XML出力・保存

メインエリアは縦に長いフォームになっており、上から順に以下のセクションが並びます。

1. **基本情報**（Original ID、名前、性別）
2. **生没年**
3. **ニスバ・ラカブ**(別名)
4. **法学派(Madhhab)・スーフィー教団**
5. **🎓 Teachers（師匠）**
6. **🧑‍🎓 Students（弟子）**
7. **📍 Activities（地理的活動）**
8. **🏛️ Institutions（所属機関）**
9. **🏅 Offices（役職）**
10. **👨‍👩‍👧 Family Relations（家族関係）**
11. **📅 Biographical Events（伝記的出来事）** ※v2.0で新設
12. **🤝 Social Relations（社会関係）** ※v2.0で新設
13. **メモ・翻訳**

## 作業の流れ

### ▶ ステップ 1：史料テキストを入力する

画面左側（サイドバー）の「📜 Source Text」エリアに、解析したい人物の伝記テキストをアラビア語で貼り付けます。

**ポイント**: 一度に入力するのは **人物一人分** のテキストにしてください。複数人を同時に入力すると、AIが混乱して正しく抽出できません。

### ▶ ステップ 2：解析する

「🔍 解析する」ボタンを押すと、Gemini APIが起動して伝記テキストを解析します。数秒〜数十秒待つと、メインエリアの各フィールドに自動的に抽出結果が反映されます。

### ▶ ステップ 3：データを確認・修正する

AIが抽出した内容を、研究者の目で確認します。間違いや漏れがあれば、各フィールドを直接編集して修正します。

**v2.0からの便利機能**:

- **▲▼ ボタン**: Teachers/Students/Activities/Institutions/Offices などの各エントリを上下に並べ替えできます。時系列順に整理する際に便利です。
- **❌ ボタン**: 不要なエントリを削除します。
- **＋ add ボタン**: 新しいエントリを追加します。
- **🗑️ 入力をクリア ボタン**: 画面右上にあり、入力内容を一括でクリアできます。誤クリック防止のため2段階確認があります。担当者名は保持されます。

### ▶ ステップ 4：XMLを確認・コピーする

メインエリアの一番下に「TEI-XML Export」セクションがあります。ここに自動生成されたXMLが表示されます。oXygen Editor などで確認したい場合は、コピーして使ってください。

**v2.0での改善**: XMLは国際的なデータベースとして利用できるよう、翻訳セクション以外は英語で統一されています。

**v3.0での変更**: 人物の `xml:id` は `AIND-{12桁の原文ID}` 形式になりました（例: `AIND-401914986553`）。Original ID を入力すれば、xml:id は自動生成されます。

### ▶ ステップ 5：スプレッドシートに保存する

最終的なデータは、サイドバーから共有スプレッドシートに保存できます。担当者名（Assignee）を選び、「💾 スプレッドシートに保存」ボタンを押します。

## 入力フィールドの詳細

### 基本情報

| フィールド | 説明 | 例 |
|---|---|---|
| **Original ID** | 史料中の元のID（12桁数字）。人物を識別する主キー。 | 401914986553 |
| Full Arabic Name | 法学派まで含めたフルネーム | إبراهيم بن أحمد ... الشافعي |
| Name Only | 本名 + 父 + 祖父 | إبراهيم بن أحمد بن عبد الكافي |
| **Sex** ※v2.0新設 | 性別 | M（男性）/ F（女性）/ U（不明） |

**v3.0での変更**: 旧版にあった「AIND-D ID」入力欄は廃止しました。xml:id は Original ID から自動派生（`AIND-{Original ID}`）するため、入力する必要はありません。XML出力時に動的に生成されます。

### 生没年（v2.0で拡張）

ヒジュラ暦の年だけでなく、年月日も入力できます。さらに、確実性と注記も記録できるようになりました。

| フィールド | 説明 | 例 |
|---|---|---|
| Birth (H) / Death (H) | ヒジュラ暦 | 850 / 850-09 / 850-09-15 |
| Birth (G) / Death (G) | 西暦（自動計算） | 1446 / 1446-12 / 1446-12-21 |
| **Cert** ※v2.0新設 | 確実性（high/medium/low/未指定） | high |
| **Note** ※v2.0新設 | 注記 | 「Ca. 850」「異説あり(901)」 |

### Teachers / Students（v2.0で大幅刷新）

各エントリは4行構成です。

**1行目**: Name（師匠/弟子の名前）+ Person ID
**2行目** ※v2.0新設:
- **Method（学習方法）**: 動詞（سمع、أخذ、قرأ など）。プルダウンから選択 or 自由記述
- **Field（学習分野）**: 学問分野（الفقه、الحديث など）。プルダウンから選択 or 自由記述
**3行目**: Text Arabic / Text Latinized / Text ID（学んだ書物）
**4行目**: Date / Place（学んだ年・場所）

▲▼ボタンで時系列順に並べ替えできます。

### Activities（v2.0で拡張）

地理的活動（移住・訪問・巡礼など）を記録します。

| Type | 説明 |
|---|---|
| born / died / buried | 出生・死亡・埋葬 |
| reside / visit / travel / study | 居住・訪問・旅行・遊学 |
| **hajj / umrah** ※v2.0新設 | 大巡礼・小巡礼 |
| other | その他 |

各エントリには日付情報（Date H / Cert / Note）も記録できます。

### Family Relations（v2.0で整理）

家族関係を記録します。relation の種類は以下の16種類です（v2.0で整理）：

- **直系**: father, mother, son, daughter
- **横系**: brother, sister, spouse
- **祖父母**: grandfather, grandmother
- **傍系**: uncle, aunt, cousin, siblings_child（甥・姪）
- **系譜**: ancestor（先祖）, descendant（子孫）
- **その他**: other

※ v2.0以前にあった p_uncle / m_uncle（父方/母方のおじ）は uncle に統合されました。
※ Family は **史料に明示的に言及された関係のみ** 記録します。nasab連鎖（محمد بن أحمد から父・祖父を自動展開）はしません。

### Biographical Events（v2.0新設）

地理的移動を伴わない人生のイベント（著作・政治事件・宗教的事件など）を記録します。

| Type | 説明 | 例 |
|---|---|---|
| political | 政治的事件 | 出陣、政争への関与 |
| cultural | 著作・知的活動 | 著作、講義 |
| religious | 宗教的事件 | 葬礼、霊的体験 |
| other | その他 | |

各エントリには日付・場所・詳細説明を記録できます。

**重要**: ハッジ（巡礼）は Activities に記録します。Bio Events には含めません。

### Social Relations（v2.0新設）

家族でも師弟でもない社会的関係を記録します。

| Type | 説明 |
|---|---|
| patron | 庇護者 |
| client | 被庇護者 |
| colleague | 同僚 |
| rival | 論敵 |
| friend | 友人 |
| correspondent | 書簡相手 |
| successor / predecessor | 後継者・前任者 |
| other | その他 |

## IDについて

このツールでは、人物・地名・機関などに外部データベースのIDを付与します。IDの種類は以下の通りです。

| 種類 | 用途 | 例 |
|---|---|---|
| GeoNames ID（数字のみ） | 地名 | 104515（メッカ） |
| Wikidata ID（Qから始まる） | 概念・組織・人物 | Q160851（ハナフィー派） |
| TMP-P-（仮ID、6桁） | まだ外部IDが見つかっていない人物 | TMP-P-000001 |
| TMP-N-（仮ID、5桁） | ニスバ | TMP-N-00001 |
| TMP-L-（仮ID、5桁） | 地名 | TMP-L-00001 |
| TMP-I-（仮ID、5桁） | 機関 | TMP-I-00001 |
| TMP-O-（仮ID、5桁） | 役職 | TMP-O-00001 |
| TMP-T-（仮ID、5桁） | 書物・典籍 | TMP-T-00001 |
| TMP-S-（仮ID、5桁） | 学習方法・分野 ※v2.0で活用 | TMP-S-00001 |

TMP-（仮ID）は後から正式なIDに差し替えることができます。

### Original ID と xml:id（v3.0で導入）

『輝く光』の各エントリには、史料テキスト内で **12桁数字の Original ID** が付与されています。これは元データに由来する不変の自然キーで、AINet-DB全体で人物を識別する主キーとなります。

TEI-XML 出力時の `xml:id` は、Original ID から機械的に派生生成されます。

| 種類 | 役割 | 例 |
|---|---|---|
| Original ID | 主キー（入力値） | `401914986553` |
| xml:id | 派生値（出力時に自動生成） | `AIND-401914986553` |

研究者が入力するのは Original ID のみで、xml:id を直接入力する必要はありません。

## ID Masterについて

画面左側の「📋 ID Master 状態」を開くと、研究グループが共有しているIDリストを確認できます。このリストに登録されている地名・機関・人物・学習方法・学習分野は、解析時にAIが自動的に正しいIDを割り当てます。

**v2.0での変更**: Method（学習方法）と Field（学習分野）のプルダウンは ID-Master から動的に語彙を読み込むようになりました。ID-Masterに新しい語彙が追加されると、アプリも自動で反映します。

## ID Masterに新規IDを登録する方法

史料を読んでいると、まだID Masterに登録されていない地名・機関・人物・書物などが出てきます。そのような場合、外部データベースでIDを調べてID Masterに追加します。

- **▶ ① 地名の場合** → GeoNames で調べる
- **▶ ② 機関・概念・法学派・スーフィー教団の場合** → Wikidata で調べる
- **▶ ③ 人物の場合** → Wikidata で調べる
- **▶ ④ 書物・典籍の場合** → Wikidata で調べる
- **▶ ⑤ 見つからない場合は仮IDのままでよい**
- **▶ ⑥ ID MasterスプレッドシートへのIDの追加方法**

## 翻訳について

解析後、画面左側の下部に「🇯🇵 日本語訳」と「🇺🇸 English」タブが表示されます。AIによる学術的な翻訳ですが、必ず原文と照らし合わせて確認してください。

## よくある問題

- **▶ 解析ボタンを押しても何も起きない**
- **▶ AIの抽出結果が明らかに間違っている**
- **▶ スプレッドシートへの保存でエラーが出る**
- **▶ ページを閉じたらデータが消えた**
- **▶ Original ID が12桁にならない**（v3.0以降の注意事項）
  - 12桁の半角数字でない場合、xml:id は派生生成されません。入力欄を修正してください。

## 作業の基本的な注意事項

- 一度に入力するのは **人物一人分** のテキストにしてください
- AIの解析結果は **必ず目で確認** してください
- 作業が終わったら **必ずXMLのコピーまたはスプレッドシートへの保存** を行ってください
- 判断に迷う場合は **Editors' Notes** に記録を残しておくと、後で見直しやすくなります

## 変更履歴

### v3.0 (2026-05-11)

データ構造の根本的な見直しを行い、人物識別子を **12桁IDベース** に統一しました。
旧 `AIND-D{連番}` 方式で発生していたID重複・順序違反・未付与といった構造的問題を解消し、
データ修正に対する耐性を大幅に向上させました。

#### 識別子方式の刷新

* xml:id を `AIND-D{5桁連番}` から `AIND-{12桁ID}` に変更
   * 旧: `xml:id="AIND-D00033"`
   * 新: `xml:id="AIND-401914986553"`
* 12桁ID(元データの自然キー)を主識別子に採用
   * データ修正があっても変動しない
   * 全15,530エントリで一意性を検証済み
* `corresp="#source_{12桁ID}"` を廃止(xml:id が役割を引き継ぐため冗長)

#### データ品質の改善

* AIND-D連番重複 48件 を解消(同じ番号が2人の別人物に振られていた問題)
* 物理順序とAIND-D順序の不整合 48件 を解消
* AIND-D未付与の立項 340件 を救済(12桁IDで自動的にxml_id付与)
* 行構造の異常 459件 を正規化(1行に複数エントリヘッダが連結していた箇所を分割)
* 13桁ID 1件 を12桁に修正
* メタヘッダ(章節見出し)に誤って付与されていたAIND-D 2件 を除去

#### アプリ(app.py)の更新

* `aind_id` フィールドを廃止し、`original_id`(12桁数字)のみを正規入力に
* `get_xml_id()` / `validate_original_id()` ヘルパーを追加
* `apply_prompt_result()` から旧形式の救済ロジックを削除
* Geminiプロンプトから `aind_id` 関連項目を削除し、`original_id`(12桁)のみ要求する形に変更
* 入力欄を12桁ID用1つに統合(xml_id 専用入力欄は削除)
* XML出力時に `xml:id="AIND-{original_id}"` を動的生成

#### 移行支援ファイル

* `step3_id_mapping_v3.tsv`: 立項13,440件のマスターリスト
* `step3_aind_to_id_changelog.tsv`: 旧AIND-D連番 → 新xml_id 対応表(XML一括置換用)
* `step3_aind_duplicates.tsv`: 重複していた旧AIND-D 48件のリスト(手動判定用)
